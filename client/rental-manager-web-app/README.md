# Rental Manager Web App

Reusable frontend baseline for the multi-tenant Rental Manager SaaS.

## Stack

- React + TypeScript + Vite
- shadcn/ui primitives
- React Hook Form + Zod
- TanStack Query for server cache
- TanStack Router file-based routing
- TanStack Table for reusable server-side tables
- Axios for HTTP transport
- Axios Mock Adapter + Faker for local API data

## Run locally

```bash
cd client/rental-manager-web-app
npm install
npm run dev
```

Open `http://localhost:5173`.

The root URL redirects to:

- `/properties` — Property list with URL pagination/search
- `/rooms` — Room list with status and rental data

Development mode enables the mock API by default, so these pages run without the .NET backend.

## Mock API

The mock layer intercepts the same Axios instance used by the real application. Components, request functions, Query hooks and route loaders do not know whether data comes from Faker or .NET.

```text
Route loader / Query hook
          ↓
request.ts
          ↓
apiClient
          ↓
Axios Mock Adapter (development)
          or
.NET API (mock disabled)
```

Configuration:

```env
VITE_API_URL=http://localhost:5008
VITE_ENABLE_MOCK_API=true
VITE_MOCK_API_DELAY=250
```

Rules:

- In development, mock mode defaults to enabled when `VITE_ENABLE_MOCK_API` is omitted.
- Set `VITE_ENABLE_MOCK_API=false` to call the real backend.
- Faker uses a fixed seed, so baseline data is repeatable.
- Property and Room mock endpoints support list, detail, create, update and delete.
- Mock validation errors use the same `.NET ProblemDetails` shape as the real API.

Mock structure:

```text
src/api/mocks/
├── factories.ts          # deterministic Faker entities
├── database.ts           # in-memory Property and Room database
├── register-handlers.ts  # endpoint handlers, filtering and pagination
└── index.ts              # start/stop mock adapter
```

## Route responsibilities

A route file owns URL concerns only:

- `validateSearch`
- path parameters
- permission/authentication checks in `beforeLoad`
- loader prefetch with `queryClient.ensureQueryData`
- selecting the page component

Example:

```tsx
export const Route = createFileRoute(
  "/_authenticated/_standard/(tenant)/rooms/",
)({
  validateSearch: (search) => searchSchema.parse(search),
  loaderDeps: ({ search }) => search,
  loader: ({ context, deps }) =>
    context.queryClient.ensureQueryData(
      roomQueries.list({
        page: deps.page,
        pageSize: deps.pageSize,
        search: deps.search || undefined,
      }),
    ),
  component: RoomListPage,
});
```

Inside a page, use the generated route API for strongly typed search state:

```tsx
const routeApi = getRouteApi(
  "/_authenticated/_standard/(tenant)/rooms/",
);

const search = routeApi.useSearch();
const navigate = routeApi.useNavigate();
```

## Layer rules

1. `components/ui` stays close to shadcn and contains no rental business rules.
2. `components/form` owns labels, errors, disabled states and React Hook Form wiring.
3. `components/common` owns reusable application patterns.
4. `api/routes/<resource>` owns server-state behavior for one resource.
5. `features/<resource>` owns business forms, columns and page composition.
6. Pages never call Axios directly.
7. Pagination, filters, sorting and search belong in URL search params.
8. Mutations invalidate or update cache inside `hooks.ts`.
9. Route loaders use `ensureQueryData` for cache reuse and prefetch.
10. Changing organization clears Query cache before rendering the new tenant.

## API resource baseline

Every resource uses the same files:

```text
api/routes/contracts/
├── types.ts       # request and response contracts
├── requests.ts    # endpoint calls only
├── queries.ts     # keys and queryOptions
├── hooks.ts       # query/mutation hooks and invalidation
└── index.ts       # public exports
```

The page consumes hooks from `index.ts`; it does not know endpoint URLs or cache invalidation details.

## Form baseline

A feature form defines only schema, defaults, fields and submit request:

```tsx
<AppForm<RoomValues>
  schema={roomSchema}
  defaultValues={defaults}
  onSubmit={(values) => createRoomMutation.mutateAsync(values)}
>
  {(form) => (
    <>
      <TextFormField
        control={form.control}
        name="code"
        label="Mã phòng"
        required
      />
      <MoneyFormField
        control={form.control}
        name="monthlyRent"
        label="Giá thuê"
        required
      />
      <FormSubmitButton>Lưu phòng</FormSubmitButton>
    </>
  )}
</AppForm>
```

Included fields:

- Text, number, date, textarea, select and switch
- Email, phone, password, money and checkbox
- Common Zod schema helpers
- `.NET ProblemDetails` field mapping
- Root server error and submit state handling

## Multi-tenant cache safety

```ts
await setOrganizationId(nextOrganizationId);
```

This stores the organization, clears Query cache and invalidates the Router. Cached data from one organization must never appear in another organization.
